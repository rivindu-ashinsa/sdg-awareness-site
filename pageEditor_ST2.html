Home Page

Technical Description

To implement the SDG 9 awareness page, key HTML tags such as <header>, <nav>, <section>, and <footer> are used to structure the content and ensure accessibility. The <header> contains the page's title and navigation links, guiding users through the site, while the <nav> tag organizes links for easy navigation. <section> tags divide the content into meaningful parts, improving readability and organization, and the <footer> tag is used for copyright and additional links. CSS properties like display: flex, justify-content, and align-items are used to create a responsive layout that adapts to different screen sizes, ensuring a mobile-friendly design. The use of background-color, font-family, and color enhances visual aesthetics, while padding and margin ensure proper spacing between elements, improving readability and user experience. By combining these HTML and CSS techniques, the page achieves a clean, accessible, and visually appealing design.

Accessibility

Accessibility has been prioritized in the design of the home page. The navigation elements, including the <a> links in the menu, are keyboard accessible, ensuring that users with motor disabilities can navigate easily using the keyboard. The <img> elements are equipped with alt attributes to provide descriptions for screen readers. Additionally, all interactive buttons, such as the menu toggle, are clearly labeled for assistive technologies using aria-label to ensure proper identification. The page has a high contrast between text and background colors to aid users with visual impairments, and text resizing is supported for better readability. The page structure is semantically correct, with the use of <header>, <nav>, <main>, and <footer> elements, which improve document flow and accessibility for screen readers. The use of proper HTML elements such as <section> and <ul> tags for lists ensures logical navigation, while the use of aria-describedby for tooltips improves the understanding of dynamic content for assistive technologies.

Table Page

Technical Description

The HTML structure is designed to showcase data on the progress of climate action for various countries, specifically targeting the UN Sustainability Goal of Climate Action. Key HTML elements used in this page include <header> for defining the page's top section with navigation and logo, <nav> for the menu navigation that allows users to access various sections like Home, Volunteer, and Feedback, and <footer> for the footer containing contact details, social media links, and subscription options. The <table> element is employed to display structured data about CO2 emissions, renewable energy use, and action plans for each country. Each <th> and <td> tag is utilized to define headers and data cells, while tooltips (data-tooltip) are included for additional information on hover. CSS is used to create a clean and responsive design, using properties like display: flex, justify-content, and align-items for layout management, and progress-bar for visualizing the percentage of renewable energy usage. Styling decisions ensure that the layout adapts to various screen sizes and that key sections like the table and footer are clearly distinguishable and user-friendly.

Accessibility

Accessibility has been a key consideration in the design of this page. The <a> elements include aria-label attributes for social media icons in the footer, ensuring that screen readers can easily identify them. All images are accompanied by alt attributes to provide descriptions for users relying on screen readers. The text contrast between the background and text has been optimized for improved readability, benefiting users with visual impairments. The navigation bar supports keyboard navigation, allowing users with motor disabilities to easily navigate through the menu. Additionally, the page's structure is semantically organized, with proper use of <header>, <nav>, <section>, and <footer> tags to improve document accessibility and understanding for assistive technologies. The table is properly structured, and tooltips are provided for more context on hover, enhancing usability for all users.

Content Page

Technical Description

The given HTML structure represents a webpage dedicated to SDG-9: Industry, Innovation, and Infrastructure. It follows a clean and responsive design approach using various HTML elements to organize and display content effectively. The <header> section contains a navigation bar with links to essential pages such as Home, Volunteer, Content, Sitemap, Table, Feedback, and Team. A menu toggle button is provided to control the visibility of the navigation links. The <main> section is organized into multiple sections, each focused on different aspects of SDG-9, such as Industry, Innovation, Infrastructure, and the way forward towards achieving the goal. The content of each section is embedded using <iframe> elements, which load external content from separate HTML files, making it easy to organize and present detailed information without cluttering the main page. For example, the first iframe loads content about 'Industry' with a focus on its significance as the backbone of economic growth, while subsequent iframes load related content on 'Innovation,' 'Infrastructure,' and 'Entrepreneurship.' The <footer> section includes company information, social media links, and an option for users to subscribe to a newsletter for updates. CSS styles such as display: flex are used to create a responsive and user-friendly layout, ensuring that the page adapts to different screen sizes. Additionally, interactive elements such as the sidebar and top-button ensure easy navigation and enhanced user experience across the site.

Accessibility

The website has been designed with accessibility in mind, featuring a navigation system that is fully keyboard-navigable, allowing users with disabilities to easily access various sections. All interactive elements, such as buttons and links, have been provided with descriptive alt text for screen reader compatibility. The sidebar menu and main content sections use clear headings, ensuring that users can easily understand the structure of the page. Additionally, all images within the content and iframes are equipped with alt attributes for screen readers. The page also ensures sufficient color contrast between text and background, improving readability for individuals with visual impairments.

Challenges and Lessons Learned

Challenges:

Throughout the development process, several challenges were encountered. One of the primary issues was ensuring cross-browser compatibility. While the pages rendered correctly in most browsers, minor styling issues were found in Internet Explorer and older versions of Safari. These inconsistencies required additional CSS fixes to maintain a consistent user experience across all platforms.

Another challenge involved optimizing the accessibility features. While basic accessibility was achieved with semantic HTML and alt text for images, ensuring comprehensive keyboard navigation and managing focus for dynamic content (like modals) proved to be more complex. This required additional JavaScript for focus management and ARIA attributes to improve screen reader compatibility, which was beyond the initial scope.

Lastly, we encountered issues with optimizing performance for mobile devices. Although the design was initially responsive, some interactive elements (like dropdown menus) didn’t perform as smoothly on touchscreens, requiring adjustments in touch event handling and padding for better touch target sizes.

Learnings:

Throughout the process, several valuable learnings were gained. First, the importance of following semantic HTML5 practices became clear, as it significantly enhanced the accessibility of the site. Using <header>, <footer>, and <main> elements improved not just accessibility, but also SEO and overall structure.

Additionally, incorporating WCAG 2.1 accessibility guidelines proved crucial. While initially challenging, integrating features like high-contrast color schemes and ensuring keyboard navigation helped create a more inclusive user experience. Implementing ARIA attributes for dynamic content (such as modals) has been a critical takeaway for ensuring screen reader users can access all content.

Finally, optimizing for mobile devices highlighted the importance of touch-friendly design elements. It was a valuable lesson to ensure all interactive components are large enough for users to interact with comfortably on small screens, reinforcing the need for mobile-first design practices.

Compliance

The Home, Table, and Content pages were designed with full adherence to modern web accessibility and usability standards, aligning with the WCAG 2.1 guidelines. We utilized semantic HTML5 elements, such as <header>, <main>, and <section>, throughout to ensure clear and machine-readable content structure. Accessibility was prioritized, with alt attributes added to all images (e.g., alt="Business Icon" on the Content page) and high-contrast color schemes (e.g., #2c3e50 on a white background) to enhance readability. Additionally, all interactive elements, including buttons and links, are keyboard-navigable, and sufficient touch targets were ensured through proper padding around clickable areas on the Home page.

The Table page was designed with proper tabular data presentation in mind, utilizing <th> headers for clarity and proper HTML structure. Furthermore, the Home page’s interactive Call-to-Action (CTA) buttons were optimized for accessibility, ensuring they meet touch target guidelines. No copyrighted content has been used, aside from placeholder images, and all assets are locally hosted, which aligns with ethical publication practices.

While we have taken every measure to ensure compliance, full accessibility also depends on the JavaScript implementation (e.g., modal.js), which should ideally include ARIA attributes for dynamic content to enhance screen reader support. At present, there is a possibility that keyboard focus management for the modal might need further refinement, and we recommend additional testing to ensure seamless navigation.

In conclusion, the pages demonstrate a strong commitment to web accessibility, usability, and ethical standards, though we acknowledge that continued testing and refinement may be needed to achieve full compliance.
